# Final Field Rules

| Rule | Status | Text |
| --- | --- | --- |
| FR01 | preserved | A public source or artifact source is required for any claim-safety label. |
| FR02 | preserved | Strong benchmark claims require split/protocol field plus baseline and metric field. |
| FR03 | preserved | Leakage claims require direct overlap/leakage testability fields; split delta alone is insufficient. |
| FR04 | revised | Replay absence caveats ordinary claims and blocks only replay-specific robustness claims. |
| FR05 | revised | Negative controls are required for strong claims and recommended for descriptive claims. |
| FR06 | preserved | Tool usefulness claims require simple baseline or ablation comparison. |
| FR07 | revised | Low-risk controls may use a minimal field profile when no strong mechanism claim is made. |
| FR08 | preserved | Cross-domain transfer claims require failed or partial cases and scope update. |
| FR09 | rejected | One uniform field list for every domain is rejected as too burdensome. |
| FR10 | preserved | Conceptual principle claims require rival principles, frozen predictions, failures, and boundaries. |
| FR11 | revised | Artifact manifests can be compact if they bind every required claim field. |
| FR12 | preserved | Not-testable is a valid outcome when required mechanism fields are absent. |
