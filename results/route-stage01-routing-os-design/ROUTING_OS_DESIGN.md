# Routing OS Design

Cross-Domain Evidence Routing OS v1 routes safe scientific and technical targets from intake to a minimum useful evidence path. It is a router, not a discovery validator.

Core flow:

- intake target descriptor
- classify target type
- reject unsafe or not-testable targets early
- plan minimum evidence route
- execute minimum evidence route where feasible
- score evidence completeness and time-to-evidence
- package public-safe route result
- promote only high-evidence targets for deeper validation
