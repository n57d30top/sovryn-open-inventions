# Data, Descriptor, and Residual Definition

- Data: safe public computational artifacts with retrievable source metadata.
- Receipt: source URL, retrieval method, timestamp, source hash where feasible, and public-safe receipt summary.
- Descriptor: simple reproducible artifact features such as schema count, line count, field completeness, package metadata completeness, or temporal ordering features.
- Residual: remaining structured discrepancy after comparing the descriptor to a simple baseline or negative control.
- Decision boundary: residual language is not allowed to strengthen the candidate when receipt stability, descriptor definition, or baseline comparison is missing.
