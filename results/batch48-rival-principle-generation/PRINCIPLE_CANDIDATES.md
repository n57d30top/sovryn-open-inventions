# Principle Candidates

| Id | Name | One-sentence claim |
| --- | --- | --- |
| P01 | Required Evidence Field Boundary | A claim cannot become stronger than the required evidence fields available to test its mechanism. |
| P02 | Claim Safety Over Metric Gain | A scientific intervention can be valuable by blocking or narrowing claims even when metrics do not improve. |
| P03 | Replay Integrity Boundary | Replay preserves evidence integrity but is not sufficient for claim validity. |
| P04 | Negative Control Generativity | Well-chosen controls generate boundary knowledge rather than merely checking pipelines. |
| P05 | Ambiguity as Active Mechanism | Ambiguity is a mechanism that changes the permissible claim set. |
| P06 | Tool Compression Boundary | Research tools compress evidence better than they discover mechanisms when simple checks dominate. |
| P07 | Metric-Only Fragility | Many evaluation failures are primarily metric-reporting failures. |
| P08 | Protocol-Claim Mismatch | Fragility emerges when claim scope exceeds protocol specificity. |
| P09 | Transfer Field Loss | Concept transfer fails when target domains lack analogous fields. |
| P10 | Repair-as-Filter | Repair acts best as a filter over claims, not as a score improver. |
| P11 | Baseline as Rival Explanation | Simple baselines are explanatory rivals, not only controls. |
| P12 | Not-Testable Boundary | Not-testable results map theory scope more honestly than forced labels. |
| P13 | Low-Risk Control Discipline | Low-risk controls prevent overgeneralization more than they validate mechanisms. |
| P14 | Evidence Packaging Dominance | Evidence packaging dominates tool value in mature benchmark programs. |
| P15 | Protocol Granularity Threshold | A protocol becomes useful only above a minimum detail threshold. |
