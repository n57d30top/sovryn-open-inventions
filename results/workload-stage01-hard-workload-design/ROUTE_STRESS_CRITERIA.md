# Route Stress Criteria

Stress criteria:

- Ambiguous target type must receive confidence and fallback route.
- Unsafe near-miss must stop without execution.
- Not-testable must require confirmation rather than one weak signal.
- Package readiness must use evidence and package thresholds.
- Deep validation promotion must require falsifier, replay feasibility, inspectability, and evidence completeness.
- Route cost must be estimated and measured.
