# Hard Workload Design

A hard workload is a mixed safe public target set where target type, route, evidence cost, and public package status are not obvious from a single keyword. The pilot uses 400 candidate descriptors and 160 blind route decisions. Hardness signals include unclear target type, missing docs, mixed repo/data/claim targets, unsafe near-misses, not-testable cases, ambiguous route choices, broken install paths, weak public metadata, duplicate targets, and irrelevant targets.

The goal is route-policy stress testing, not broad acceleration proof.
