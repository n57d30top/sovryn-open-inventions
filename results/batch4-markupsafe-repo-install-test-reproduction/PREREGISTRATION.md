# Preregistration

Question: Can an executed baseline, audit, or challenger produce useful external research evidence for pallets/markupsafe GitHub repository?

Metrics were fixed before reporting: row/test counts where applicable, baseline metrics, challenger metrics, wins/losses/ties, missingness or duplicate rates where applicable, and execution success/failure.

Kill criteria: reject or downgrade any challenger that fails to beat the best simple baseline under the preregistered metric, fails execution, leaks holdout information, or lacks concrete metrics.
