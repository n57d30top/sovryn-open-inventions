# Learning Across Results

The program shows learning across results in four concrete ways.

## 1. Execution changed the workflow

Batch 5 showed that smaller, deeper execution-backed results were more useful than shallow count-filling. Batch 6 used that lesson by requiring explicit reproduction ladder levels.

## 2. Reproduction findings created tool pressure

Batch 6 dataset and baseline findings exposed repeated needs: source-bound dataset checks, baseline comparisons, and public-safe execution summaries. Batch 7 responded by building small tools instead of adding framework features.

## 3. Tool claims were tested instead of trusted

Batch 7 tools were not assumed reusable. Batch 8 tested them on six new targets. The schema tool was narrowed, the metric tool was promoted, and the pytest summary tool was kept narrow.

## 4. The next direction was selected from evidence

The next direction is dataset and metric validation because that line had the best combination of execution success, reusable tools, safe public data, baseline comparisons, and useful negative findings.
