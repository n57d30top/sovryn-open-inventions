# Policy Lessons

Failure and weakness classes extracted from the previous run:

1. Ambiguous targets need confidence scoring.
2. Mixed-signal targets need fallback routes.
3. Static metadata can overstate package readiness.
4. Not-testable needs confirmation.
5. Unsafe near-misses need early stop.
6. Deep promotion needs stricter thresholds.
7. Cost should be estimated before execution.
8. Route errors should be first-class outputs.
9. Package readiness should not equal evidence completeness.
10. Time-to-evidence claims need failure penalties.
11. Low-risk controls should stay in the workload.
12. Public artifacts need explicit no-overclaim language.
