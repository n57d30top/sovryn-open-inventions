# Preregistration

Research question: Can a source-card parser recover expected HotpotQA evidence fields more completely than a keyword-only baseline from public metadata?

Metrics fixed before evaluation:
- expected field count
- keyword baseline field recall
- source-card parser field recall
- recorded limitations

Kill criteria:
- dataset card inaccessible
- expected evidence fields absent
- parser claim extended to model QA performance

Publication criteria: publish only if the result includes external target binding, concrete metrics, failures/losses, limitations, reproducibility instructions, and public hygiene pass.
