# Rejected or Reserved Domains

- materials/property residuals: public targets exist, but API/key and domain tooling burden is high for this wave
- benchmark protocol failures: kept as reserve to avoid overfitting to prior benchmark-repair work
- dataset reliability anomalies: kept as supporting control, not selected as primary new domain
- protein structure metadata regularities: safe, but larger parser/tooling burden than this search wave
- scientific simulation reproducibility: safe, but deeper repo/toolchain work needed
- open-source scientific documentation/protocol drift: useful audit domain, but weak discovery-candidate yield
- graph/network anomaly mechanisms: safe, but public target selection was less mature than selected domains
