# Examples and Non-Examples

## Examples

- HAR: source/subject-aware repair downgrades random-only interpretation.
- Optical Digits: file/source split should be used before random split claims.
- Shuttle: rare-class metric reporting narrows accuracy-only claims.
- Vehicle: ambiguity gate blocks strong protocol-first claims.
- Letter: low-risk control shows duplicate repair may be unnecessary.

## Non-examples

- A protocol-absent sklearn dataset should not be described as protocol-repaired.
- Duplicate repair should not be claimed when no cross-split duplicate features are found.
- Ambiguity gates do not reconstruct missing source instructions.
- A metric-only change is not evidence of better model performance.
- A replay hash check is not cross-domain validation.
