# Candidate Concepts

- concept-evaluation-fragility: Evaluation Fragility (survive). A claim is fragile when evaluation protocol changes alter the conclusion.
- concept-protocol-sensitivity: Protocol Sensitivity (survive). A target's metrics materially depend on source-described split protocol.
- concept-split-induced-optimism: Split-Induced Optimism (downgrade). Random splits make a result look better than source protocols.
- concept-replay-robustness: Replay Robustness (downgrade). A claim remains stable under replay or fresh execution.
- concept-ambiguity-barrier: Ambiguity Barrier (downgrade). Protocol ambiguity blocks strong evaluation claims.
- concept-metric-illusion-zone: Metric Illusion Zone (reject). Metric choice hides class-specific weakness.
- concept-protocol-leakage-shadow: Protocol Leakage Shadow (reject). A split delta resembles leakage but lacks mechanism evidence.
- concept-tool-value-compression: Tool Value Compression (downgrade). Custom tool value narrows to packaging when simple baselines dominate.
- concept-evidence-packaging-dominance: Evidence Packaging Dominance (reject). Evidence curation adds value while raw discovery value is low.
- concept-benchmark-claim-fragility: Benchmark Claim Fragility (downgrade). Benchmark claims weaken when protocol and controls are added.
- concept-control-anchored-theory: Control-Anchored Theory (downgrade). A theory requires low-risk controls to avoid cherry-picking.
- concept-ambiguity-weighted-confidence: Ambiguity-Weighted Confidence (reject). Confidence is reduced in proportion to unresolved protocol ambiguity.
