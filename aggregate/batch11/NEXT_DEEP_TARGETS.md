# Next Deep Targets

Candidates for a future deep target:

1. A classification benchmark with an official train/test split and documented source protocol.
2. A dataset with known group or acquisition metadata so split leakage can be tested directly.
3. A medium-sized tabular target where duplicate-aware evaluation can materially change estimates.
4. A repo-backed benchmark where `pytest_repro_summary` can be checked against runtime collection.
5. A target with published baseline scripts rather than only dataset metadata.
